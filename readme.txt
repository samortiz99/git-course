Explanation for each git function

$ pwd = se usa para seleccionar una ubicación
$ ls = se usa para ver que archivos hay dentro de la dirección actual

$ git init = para inicializar un nuevo proyecto
$ git status = para ver el estado de los archivos
$ git add <filename> = para agregar un archivo al staging area, si se quieren
  añadir todos los cambios, entonces se usa un punto <.>
$ git commit = para crear un primer punto de control de nuestro code
--después de usar commit, para volver al editor se usa ":wq" que significa write quit
$ git log = para ver todos los commits que hemos creado
$ git checkout = para revertir los cambios de los archivos
$ git diff = to view in console, the changes made in files
-- para ignorar archivos, se crea un archivo llamado <.gitignore>, donde escribiremos
   adentro de el el nombre de los archivos que queremos ignorar para que no se muestren
   en la consola de git
$ git commit -m "<comment>" = es la función de $ git commit resumida para no entrar
  en el otro editor de código
$ git branch = para ver o crear una nueva carpeta
